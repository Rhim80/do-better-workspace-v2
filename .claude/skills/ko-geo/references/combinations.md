# 모듈 조합 시너지 & 적용 순서 가이드

논문 §5.3 *Combination of GEO Strategies* + Figure 4 (heatmap) 기반.
실제 변환 시 어떤 모듈을 어떤 순서로 묶어 적용할지에 대한 권장.

## 1. 검증된 조합 시너지 (논문 Figure 4)

논문은 상위 4개 메서드(Cite Sources, Fluency, Statistics, Quotation)의 모든 쌍 조합을 GEO-bench 일부(200개 샘플)에서 평가했다.

| 조합 | PAWC 평균 향상률 | 비고 |
|------|------------------|------|
| **Fluency + Statistics** | **+35.8%** | 단일 최강 조합. 단일 최고치 대비 +5.5%↑ |
| Fluency + Cite Sources | +34.4% | Cite Sources가 단독으론 약하지만 Fluency와 묶이면 강해짐 |
| Fluency + Quotation | +33.0% | 안정적 |
| Statistics + Quotation | +35.4% | 가장 인기 있는 "수치 + 인용" 조합 |
| Statistics + Cite Sources | +30.3% | 사실 기반 도메인 표준 조합 |
| Quotation + Cite Sources | +20.1% | 시너지가 가장 작음 (둘 다 "타인의 권위" 의존) |

### 핵심 함의 (논문이 직접 언급)

- **Fluency Optimization은 다른 모든 모듈과 시너지가 좋다** (heatmap 우측 열 평균이 가장 높음, Avg 31.4%)
- **Cite Sources는 단독 효과는 작지만 다른 모듈과 결합 시 평균 31.4%까지 상승** — 단독 사용보다 조합으로 활용
- **GEO는 조합 사용이 현실적** — 콘텐츠 제작자는 보통 여러 모듈을 동시에 적용한다

## 2. KoGEO 권장 조합 패턴

### 패턴 A — "신뢰 기반" (사실/스펙 도메인용)

```
M1 통계 + M3 출처 + M4 유창성
```

- 적용 도메인: 법·정부, 의료, 금융, 제품 스펙, 학술
- 예상 PAWC 향상: +30~38%
- 적용 순서: M1 → M3 → M4

### 패턴 B — "권위 기반" (해설/인물/역사 도메인용)

```
M2 인용구 + M5 두괄식 + M4 유창성
```

- 적용 도메인: 인물, 역사, 사회 해설, 토론
- 예상 PAWC 향상: +35~42% (Quotation 단독 41% 기반)
- 적용 순서: M5 → M2 → M4

### 패턴 C — "B2C 트래픽" (마케팅/리뷰 도메인용)

```
M5 두괄식 + M1 통계 + M10 FAQ + M4 유창성
```

- 적용 도메인: 맛집, 뷰티, 여행, 쇼핑 리뷰
- 예상 PAWC 향상: +28~35%
- 적용 순서: M5 → M1 → M10 → M4 → M9 (마지막 정제)

### 패턴 D — "저순위 사이트 부스트" (Rank 4-5)

```
M3 출처 + M2 인용구 + M1 통계 + M4 유창성
```

- 적용 조건: 추정 SERP 순위 4위 이하
- 예상 PAWC 향상: +50%↑ (Rank-5 Cite Sources +115% 기반)
- 논문 Table 2 근거: Rank-5에서 Cite Sources +115%, Quotation +99%, Statistics +97%
- 적용 순서: M3 → M1 → M2 → M4 → M9

### 패턴 E — "토론·정책" (논쟁성 콘텐츠)

```
M8 권위적 + M1 통계 + M3 출처 + M5 두괄식
```

- 적용 도메인: Debate, 정책 칼럼, 시사
- 예상 PAWC 향상: +25~32%
- 적용 순서: M5 → M1 → M3 → M8 → M4

## 3. 표준 적용 순서 (디폴트)

도메인이 명확하지 않거나 사용자가 지정하지 않은 경우 다음 순서로 적용한다.

```
1. M5 두괄식 배치        — 구조 먼저 정리
2. M1 통계 보강          — 정량 신호 추가
3. M2 인용구 삽입        — 권위 신호 추가
4. M3 출처 표기          — 검증 가능성 추가
5. M6 전문 용어 OR M7 쉬운 표현  — 도메인에 따라 택1
6. M10 FAQ 블록 (선택)   — B2C/롱테일 쿼리 노릴 때
7. M8 권위적 어조 (선택) — 토론·역사·논쟁성일 때만
8. M4 유창성 향상        — 다른 모듈 적용 후 다듬기
9. M9 키워드 과밀 회피   — 최종 정제
```

### 순서 결정 원리

- **구조부터 콘텐츠로**: M5(구조) → M1~M3(콘텐츠 추가) → M4(다듬기) → M9(정제)
- **추가 모듈을 먼저, 정제 모듈을 나중에**: 새 텍스트가 들어간 뒤에 유창성·키워드 점검을 해야 함
- **선택 모듈은 도메인 판단 후**: M6/M7/M8/M10은 도메인에 따라 적용 여부 결정

## 4. 함께 쓰면 안 되는 조합

### 충돌 조합

| 조합 | 이유 |
|------|------|
| M6 전문 용어 + M7 쉬운 표현 | 정반대 방향. 한 콘텐츠 안에서 일관된 톤 필요 → 도메인 따라 택1 |
| M8 권위적 + B2C 친근 톤 | 마케팅 콘텐츠에 권위적 어조는 거리감을 늘려 클릭률 저해 |
| 통계 보강 + 키워드 스터핑 잔존 | M9를 적용 안 하면 통계 보강이 키워드 반복을 가릴 뿐 -8% 역효과 누적 |

### 비권장 조합

- **M2 + M3을 단독으로만**: 시너지가 가장 작음 (+20%). M4 또는 M1과 함께.
- **너무 많은 모듈 동시 적용 (7개+)**: 길이만 부풀고 핵심 결론이 흐려진다. 3~5개가 최적.

## 5. 효과 측정 방법

KoGEO 변환 효과를 사용자가 실측하려면:

### A. Perplexity / ChatGPT 검증

1. 변환 전: 타깃 쿼리 5개를 골라 Perplexity·ChatGPT에 검색 → 자기 콘텐츠 인용 여부 기록
2. 변환 후 1~2주: 같은 쿼리로 재검색 → 인용 여부 + 인용 위치(전반/중반/후반) 기록
3. PAWC 근사: (인용된 쿼리 수 × 응답 내 위치 가중치) 합산

### B. 자기 점검

- M1: 정량 수치 개수 (변환 전/후)
- M2: 인용구 개수
- M3: 출처 표기율 (핵심 주장 대비)
- M5: 첫 100자 내 결론 유무
- M9: 최다 반복 키워드의 등장 횟수

### C. 도구 활용

- 검색 모니터링: 자체 트래픽 분석에서 "Perplexity", "ChatGPT", "Gemini" 리퍼러 추적
- 텍스트 분석: 한국어 형태소 분석기로 키워드 중복도 측정

## 6. 재검증 권고 주기

- **콘텐츠 적용 직후**: 1~2주 후 1차 측정
- **분기별**: 생성형 엔진 모델·랭킹 알고리즘 변경 가능성 → 분기별 재진단
- **새 모델 출시 시점**: GPT/Claude/Gemini 메이저 업데이트 후 재측정 권장 (논문도 §9 Limitations에서 명시)

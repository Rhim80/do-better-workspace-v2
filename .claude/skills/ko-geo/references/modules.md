# KoGEO 10개 변환 모듈 상세 프롬프트

논문 *"GEO: Generative Engine Optimization"* (Aggarwal et al., KDD 2024, arXiv:2311.09735) 기반.
각 모듈 적용 시 이 파일의 정의·프롬프트·체크리스트를 따른다.

성능 수치는 논문 GEO-bench(10K 쿼리, gpt-3.5 기반 GE) 결과의 상대 향상률 (Relative Improvement) 이다.

---

## M1 — 통계 보강 (Statistics Addition)

- **논문 메서드**: Statistics Addition
- **PAWC**: +30% / **Subjective Impression**: +37% (가장 큰 폭)
- **원리**: 정량 수치는 AI가 신뢰 신호로 채택. 정성 표현보다 우선 인용된다.

**적용 프롬프트:**

```
콘텐츠의 정성적 표현을 정량적 수치로 교체·보강하세요.
- "많은" → "월 방문객 약 X명 기준"
- "빠르게" → "전년 대비 X% 증가"
- "대부분" → "응답자의 X% (출처)"
- "오래된" → "OO년 운영 (창립 19XX년)"
실제 데이터가 없으면 "(추정)"을 명시하고 합리적 범위 내 수치를 사용한다.
출처 형식: (기관명, 연도) 또는 [출처명].
도메인 내에서 검증 가능한 수치를 우선한다.
```

**체크리스트:**
- [ ] 한 문단당 최소 1개 정량 수치
- [ ] 모든 수치에 출처 또는 "(추정)" 표기
- [ ] 단위(원, %, 명, 분, kg 등)를 명확히

**도메인 적합도:** Law·정부 ★★★, 오피니언 ★★★, 뷰티/헬스 ★★★, 맛집 ★★

---

## M2 — 인용구 삽입 (Quotation Addition)

- **논문 메서드**: Quotation Addition
- **PAWC**: +41% (전체 1위) / **Subjective Impression**: +28%
- **원리**: 전문가/공신력 있는 발언은 AI가 인용 근거로 채택. 단일 전략 중 가장 강력.

**적용 프롬프트:**

```
신뢰할 수 있는 전문가나 공신력 있는 기관의 발언을 인용구로 추가하세요.
형식:
> "인용 내용"
> — 이름/직함 (소속, 연도)

규칙:
- 실존하는 전문가/기관만 사용 (날조 금지)
- 해당 도메인 권위자 (의사·연구원·교수·기관 대변인 등)
- 인용 내용이 콘텐츠 주제와 직결
- 길이는 1~3문장
- 인용구는 관련 본문 바로 뒤에 배치
- 한 콘텐츠당 1~2개가 자연스러움
```

**체크리스트:**
- [ ] 출처가 검증 가능한가 (실존 인물·기관)
- [ ] 인용 내용이 본문 주장과 일치하는가
- [ ] 형식이 통일되었는가 (`> "..." — 이름(연도)`)

**도메인 적합도:** People & Society ★★★, 역사 ★★★, 인물·해설 ★★★, 토론 ★★

---

## M3 — 출처 표기 (Cite Sources)

- **논문 메서드**: Cite Sources
- **PAWC**: +27% / **순위 5위 사이트에서 +115.1%** (논문 Table 2)
- **원리**: 검증 가능한 출처는 응답에 포함시킬 정당성을 부여. **저순위·신규 사이트 가시성 향상에 가장 강력**.

**적용 프롬프트:**

```
주요 사실·주장에 신뢰할 수 있는 출처를 추가하세요.
형식: "사실 내용 (기관명, 연도)" 또는 "사실 내용 [출처명]"
규칙:
- 정부·대학·연구소·언론사·국제기구 같은 공신력 있는 기관 우선
- URL 전체보다 "기관명+연도" 형식
- 이미 출처가 있으면 그대로 유지하고 없는 곳에만 추가
- 한 문장 안에 출처 2개 초과 금지 (가독성)
- 한국 도메인 콘텐츠는 통계청·식약처·KOSIS·언론사·학회 우선
```

**체크리스트:**
- [ ] 핵심 주장 80% 이상에 출처가 있는가
- [ ] 추측/의견 문장은 명시적 표시 ("개인적 견해", "추정")가 있는가
- [ ] 출처 형식이 일관된가

**도메인 적합도:** Law·Gov ★★★, 사실 기반 ★★★, 뉴스/리포트 ★★★. **저순위 콘텐츠일수록 1순위로 적용.**

## M4 — 유창성 향상 (Fluency Optimization)

- **논문 메서드**: Fluency Optimization
- **PAWC**: +28% / Statistics와 결합 시 단일 최강 조합 (+5.5%↑)
- **원리**: 매끄러운 텍스트는 AI 요약·재구성 시 손실이 적다. 다른 모듈과의 시너지가 가장 좋음.

**적용 프롬프트:**

```
어색한 문장·번역체·반복을 자연스러운 한국어로 다듬으세요.
- 비문 교정 (주어-서술어 호응, 시제 일관성)
- 번역체 제거 ("~할 수 있다고 알려져 있다" → "~한다")
- 군더더기 제거 (잉여 부사·중복 의미 단어)
- 호흡 조절 (한 문장 60자 초과 시 분할)
- 단락 간 흐름 (접속사 자연화)
원본의 정보·의미·뉘앙스는 유지한다.
```

**체크리스트:**
- [ ] 한 문장 평균 길이 40~70자
- [ ] 한 문단 3~6문장
- [ ] 번역체·문어체 잔존 0
- [ ] 능동/수동, 시제 일관

**적용 시점:** 다른 모듈 적용 후 **최종 다듬기 단계**에서 실행. 가장 마지막 또는 마지막 직전.

**도메인 적합도:** Business ★★★, Science ★★★, Health ★★★, **모든 도메인 공통 권장**.

---

## M5 — 두괄식 배치 (Position Front-loading)

- **논문 근거**: PAWC는 위치별 가중치를 지수 감쇠 함수 e^(-pos/|S|)로 적용. 앞부분 인용일수록 가중치 큼.
- **원리**: AI는 응답 앞쪽 정보를 우선 채택. 핵심 결론·수치를 본문 상단으로 끌어올린다.

**적용 프롬프트:**

```
콘텐츠를 역피라미드 구조(결론 → 근거 → 상세)로 재구성하세요.
- 첫 문단: 핵심 결론·요점 1~2문장 (그 자체로 답이 되도록)
- 둘째 문단: 핵심 근거·주요 수치
- 이후: 세부 내용·맥락·예시·부연

추가로 글머리 "핵심 요약" 블록을 최상단에 둔다:
**핵심 요약**
- [핵심 포인트 1: 가장 중요한 사실/결론]
- [핵심 포인트 2: 두 번째 중요 사실]
- [핵심 포인트 3: 실용적 조언/차별점]

각 줄은 다른 문장 없이도 단독으로 의미가 통해야 한다.
원본의 모든 정보를 유지하되 순서만 재배치한다.
```

**체크리스트:**
- [ ] 첫 100자 안에 결론이 등장하는가
- [ ] 핵심 요약 3줄이 독립적으로 의미를 전달하는가
- [ ] 본문이 결론을 뒷받침하는 흐름인가

**도메인 적합도:** **모든 도메인 공통 권장**.

## M6 — 전문 용어 보강 (Technical Terms)

- **논문 메서드**: Technical Terms
- **PAWC**: +18~21% (중간 효과)
- **원리**: 도메인 전문 용어는 AI가 권위 있는 출처로 인식하는 신호. 단, 과도하면 가독성 저하.

**적용 프롬프트:**

```
일반적 표현을 해당 분야 전문 용어/개념어로 교체·보강하세요.
- 단순 설명 → 개념어 + 간략 정의
- 영어 전문 용어는 한국어(영어) 병기
- 예: "피부 보호" → "피부 장벽(Skin Barrier) 강화"
- 예: "검색 최적화" → "SEO(Search Engine Optimization)"
- 예: "심전도 검사" → "심전도(ECG) 검사"

균형 원칙:
- 단락당 신규 전문 용어 1~2개로 제한
- 처음 등장 시 (영어/약어/정의)를 1회 병기, 이후 반복 안 함
- 일반 독자가 5초 안에 이해 못 할 용어는 짧은 설명 동반
```

**체크리스트:**
- [ ] 처음 등장한 전문 용어에 영문 병기 또는 한 줄 정의가 있는가
- [ ] 가독성이 원본 대비 크게 떨어지지 않는가
- [ ] 도메인과 무관한 용어 남용은 없는가

**도메인 적합도:** IT/기술 ★★★, 헬스/뷰티 ★★, 법·정부 ★★, 맛집 ★ (소량만)

---

## M7 — 쉬운 표현 (Easy-to-Understand)

- **논문 메서드**: Easy-to-Understand
- **PAWC**: +14~22% (M6과 정반대 방향이지만 둘 다 효과 있음)
- **원리**: AI는 명료한 텍스트를 인용·재구성하기 쉽다. 일반 독자 대상 콘텐츠에 강력.

**적용 프롬프트:**

```
어려운 표현·전문 용어가 많은 구간을 일반 독자가 이해할 수 있도록 풀어쓰세요.
- 한자어 → 일상어 (예: "유의미한" → "의미 있는")
- 긴 수식어 분리 (한 문장 안에 형용사구 3개 이상이면 분할)
- 추상 개념은 구체 예시 1개 동반
- 비유/은유 활용 (단, 도메인에 어울리는 범위)

주의: 
- M6(전문 용어 보강)과 동시 적용 시 충돌 가능. 도메인이 일반 독자 타깃이면 M7 우선.
- 학술/리포트 톤이 필요하면 M6 우선.
```

**체크리스트:**
- [ ] 중학생도 이해할 수준인가 (B2C 콘텐츠 기준)
- [ ] 구체 예시·비유가 적절히 들어갔는가
- [ ] 정보가 단순화 과정에서 손실되지 않았는가

**도메인 적합도:** 일반 소비자 콘텐츠 ★★★, B2C 마케팅 ★★★, 교육 ★★★. **M6와 택1.**

## M8 — 권위적 어조 (Authoritative)

- **논문 메서드**: Authoritative
- **PAWC**: +1~2% (단독 효과는 매우 약함, baseline 수준) / **Debate 도메인에서만 1위**
- **원리**: 단순히 "권위적"이게 쓴다고 가시성이 오르지는 않는다. **토론·역사·논쟁성** 콘텐츠에서만 선택 적용.

**적용 프롬프트:**

```
[적용 조건: 콘텐츠가 정책 토론·역사 분석·논쟁성 주장 등 단호한 어조가 자연스러운 도메인일 때만]

문체를 더 단호하고 설득력 있게 다듬으세요.
- 약한 표현 제거 ("것 같다", "할 수도 있다", "아마도") → 단정·근거 기반 진술
- 1인칭 약화, 객관적 시점 강화
- 능동태·간결한 단언
- 단, 사실이 아닌 부분에는 적용 금지 (의견/주장 영역에만)

주의: 광고성·B2C 친근 콘텐츠에는 적용하지 않는다 (역효과).
```

**체크리스트:**
- [ ] 콘텐츠가 토론·역사·정책·전문 분석 도메인인가? (아니면 적용 보류)
- [ ] 의견·주장 영역에만 적용했는가
- [ ] 단정 표현이 사실 왜곡으로 이어지지 않았는가

**도메인 적합도:** Debate ★★★, History ★★, Science ★★. 그 외 도메인은 **건너뛸 것**.

## M9 — 키워드 과밀 회피 (Anti-Keyword-Stuffing)

- **논문 근거**: Keyword Stuffing은 PAWC -8%, Subjective Impression 사실상 무변화 (논문 Table 1).
- **원리**: SEO 시대 관행 (키워드 반복 삽입)은 GEO에서 **역효과**. AI는 부자연스러운 반복을 신뢰도 저하 신호로 처리.

**적용 프롬프트:**

```
동일 키워드 또는 동의어 군이 3회 이상 반복되는 구간을 자연스럽게 분산시키세요.

교체 방법:
- 동의어 사용 ("강아지" ↔ "반려견" ↔ "애완견")
- 대명사·지시어 활용 ("이", "그", "해당", "위")
- 상위·하위 개념어로 대체 ("화장품" ↔ "스킨케어 제품")
- 문장 재구조화 (능/수동 전환, 주어 변경)

목표: 텍스트 전체 유창성(Fluency) 향상. 반복 강도가 자연스러운 변주 수준에 머무르도록.
```

**체크리스트:**
- [ ] 한 문단 안에 동일 명사 3회 이상 반복 0
- [ ] 제목·소제목 키워드가 본문 첫 100자에 자연스럽게 1회만 등장
- [ ] 메타 디스크립션·HTML alt 등 보조 텍스트에도 적용

**적용 시점:** 다른 모듈 적용 후 **최종 정제 단계**에서 실행 (M4 직후 또는 직전).

**도메인 적합도:** **모든 도메인 공통 필수 점검**.

## M10 — FAQ 블록 (Query-Variant Matching)

- **논문 직접 메서드 아님**: 논문 2.1에서 GE가 사용자 쿼리를 여러 서브쿼리로 reformulate함을 명시. FAQ 형식은 이 reformulation 변형 패턴과 직접 매칭되어 인용 후보군에 포함될 확률을 높인다.
- **원리**: 한 콘텐츠가 여러 자연어 쿼리에 매칭되면 더 다양한 응답에 인용된다.

**적용 프롬프트:**

```
콘텐츠를 바탕으로 사용자가 AI에게 물을 만한 질문 4~5개와 답변을 추가하세요.

규칙:
- 질문은 실제 사용자 언어로 자연스럽게 ("XX가 뭔가요?", "XX 효과 있나요?", "XX 얼마나 걸려요?")
- 각 답변은 2~3문장으로 핵심만
- 콘텐츠의 서로 다른 측면을 다루는 질문 선택 (가격/효과/시간/방법/주의)
- 한국어 음성 검색 패턴 반영 (긴 질문, 구어체)

형식:
**자주 묻는 질문**

**Q. 질문 내용?**
A. 답변 내용.

**Q. 다음 질문?**
A. ...
```

**체크리스트:**
- [ ] 4~5개 질문이 각각 다른 측면을 다루는가
- [ ] 질문이 실제 검색 쿼리처럼 자연스러운가
- [ ] 답변이 본문 내용과 일관되며 단독으로도 의미가 통하는가

**도메인 적합도:** B2C 서비스/제품 ★★★, 맛집·여행 ★★★, 소상공인 ★★★, 교육 ★★. B2B 기술 백서 ★ (단, 기술 FAQ 섹션은 효과 큼).

## 모듈 효과 요약 (논문 GEO-bench Absolute Impression, Table 1 기준 정리)

## 적용 시 흔한 실수

1. **모든 모듈을 다 적용하려는 경향** → 도메인에 맞는 3~5개 선택이 효과적. 모든 모듈 적용은 길이만 부풀린다.
2. **통계·인용을 만들어내는 것** → 논문은 사실 보존을 전제로 한 텍스트 변환을 다룬다. 날조는 사용자 신뢰 + 법적 리스크.
3. **M6과 M7을 동시 적용** → 서로 반대 방향. 도메인·타깃 독자에 따라 택1.
4. **M8을 마케팅 콘텐츠에 적용** → 친근한 어조가 필요한 B2C에서는 역효과.
5. **M9를 마지막에 안 거치고 종료** → 다른 모듈 적용 중 키워드 반복이 늘어날 수 있다. 반드시 마지막에 점검.

# GEO 평가 지표 — Position-Adjusted Word Count & Subjective Impression

논문 *"GEO: Generative Engine Optimization"* (Aggarwal et al., KDD 2024) §2.2.1 기반 정리.
이 스킬에서 점수·향상률을 계산할 때 참조하는 지표 정의.

---

## 1. Generative Engine 구조

논문은 생성형 엔진(GE)을 다음과 같이 형식화한다:

```
사용자 질의 q_u  →  Query Reformulation Model  →  서브쿼리 집합 {q_1, ..., q_n}
                                              ↓
                                          Search Engine
                                              ↓
                                       소스 집합 S = {s_1, ..., s_m}
                                              ↓
                                        Summarizing Model
                                              ↓
                                        Response Model
                                              ↓
                                    응답 r (인라인 인용 포함)
```

핵심: GE는 검색엔진 결과를 **요약·재구성**해서 응답한다. 따라서 콘텐츠가 인용되려면 단순 키워드 매칭이 아닌 "요약·재구성에 적합한 형태" 여야 한다.

이는 GEO가 SEO와 본질적으로 다른 이유다. SEO는 랭킹 신호(백링크·도메인 권위)에 영향받지만, GEO는 콘텐츠 텍스트 자체의 구조·신뢰도·인용 적합성에 영향받는다.

---

## 2. Position-Adjusted Word Count (PAWC)

응답 r 안에서 인용 c_i가 차지하는 "비중"을 위치 가중치를 적용해 측정한다.

### 정의 (논문 식 3을 풀어 쓴 것)

```
Imp_PWC(c_i, r) = Σ_{s ∈ S_ci} (|s| · e^(-pos(s)/|S|))  /  Σ_{s ∈ S_r} |s|
```

기호:
- S_ci : 응답 r 안에서 인용 c_i를 사용하는 문장들의 집합
- S_r  : 응답 r의 전체 문장 집합
- |s|  : 문장 s의 단어 수
- pos(s): 응답 안에서 문장 s의 위치 (앞쪽일수록 작은 값)

### 직관

1. **단어 수 비중**: 인용된 문장이 응답에서 차지하는 단어 수가 많을수록 PAWC ↑
2. **위치 가중**: 응답 앞에 등장할수록 가중치 ↑ (지수 감쇠 e^(-pos/|S|))
3. **출처 분배**: 한 문장이 여러 인용을 동시에 사용하면 단어 수를 균등 분배

### 함의 (이 스킬에 반영된 점)

- **앞쪽에 핵심 정보를 둘수록 가중치가 커진다** → M5 두괄식 배치
- **응답에 더 많이 인용되어야 한다** → M1 통계, M2 인용구, M3 출처 (인용 가치를 높이는 모듈)
- 단순히 콘텐츠를 길게 늘리는 것은 효과 없음 (분모도 같이 커짐)

## 3. Subjective Impression (SI)

객관 지표인 PAWC만으로는 사용자가 인용을 어떻게 받아들이는지 측정할 수 없다. 논문은 G-Eval (GPT 기반 평가) 방식으로 7개 서브 지표를 평가한다.

### 7개 서브 지표

| 서브 지표 | 정의 | KoGEO 모듈과 연결 |
|----------|------|-------------------|
| **Relevance** (관련성) | 인용 문장이 사용자 질의와 얼마나 직접 관련있는가 | M3 출처, M1 통계, M5 두괄식 |
| **Influence** (영향력) | 응답이 해당 인용에 얼마나 의존하는가 | M2 인용구, M3 출처 |
| **Uniqueness** (고유성) | 인용이 다른 출처에 없는 정보를 제공하는가 | M2 인용구, M1 통계 |
| **Subjective Position** (주관적 위치) | 인용이 응답에서 두드러져 보이는가 | M5 두괄식, M4 유창성 |
| **Subjective Count** (주관적 양) | 인용 정보의 양이 충분히 많아 보이는가 | M1 통계, M3 출처, M10 FAQ |
| **Probability of Click** (클릭 확률) | 사용자가 출처를 클릭할 만한가 | M2 인용구, M3 출처, M5 두괄식 |
| **Diversity** (다양성) | 응답 내 다양한 측면을 커버하는가 | M10 FAQ, M1 통계 |

### 함의

- **PAWC만 높이면 안 된다**: 사용자 인지 단계의 영향(클릭·신뢰)도 함께 챙겨야 한다.
- **Statistics Addition은 SI 1위 (+37%)**: 정량 수치는 객관 지표뿐 아니라 사용자 인지에서도 가장 강력한 신호.
- **Quotation Addition은 PAWC 1위 (+41%)지만 SI는 +28%**: 객관적으론 가장 많이 인용되지만, 사용자가 클릭·신뢰하는 데에는 통계가 더 효과적.

## 4. KoGEO 진단(geo_analyze) 점수 산출 방식

각 모듈을 0~10점으로 평가할 때 다음 두 축을 고려한다:

```
모듈 점수 = (1 - 결핍도) × 10
결핍도   = (논문이 권장하는 양 - 현재 콘텐츠의 해당 요소 양) / 권장량
```

예시:

| 모듈 | 점검 항목 | 만점(10점) 기준 |
|------|----------|----------------|
| M1 통계 | 정량 수치 개수, 출처 표기율 | 1000자당 정량 수치 3개 이상 + 출처 80% |
| M2 인용구 | 전문가/기관 인용 개수 | 콘텐츠당 1~2개, 도메인 적합 |
| M3 출처 | 핵심 주장에 대한 출처율 | 핵심 주장 80%에 출처 |
| M4 유창성 | 비문·번역체·중복 어휘 비율 | 비문 0, 한 문장 평균 40~70자 |
| M5 두괄식 | 첫 100자에 결론, 핵심 요약 블록 유무 | 둘 다 만족 |
| M6/M7 | 도메인 적합 용어 비율 | 도메인 타깃에 일치 |
| M8 권위적 | 도메인 적합 여부 | Debate/History이고 단정 표현 적정 |
| M9 키워드 과밀 | 동일 키워드 3회 이상 반복 구간 수 | 0개 |
| M10 FAQ | FAQ 섹션 유무, 질문 수 | 4~5개 |

**전체 점수 = Σ 모듈 점수 (100점 만점)**

**예상 인용률 향상 추정**:
- 적용할 모듈 각각의 PAWC 향상치를 합산하되, 중복·시너지 보정 적용
- 단순 합이 아니라 **최대 +40% 캡** (논문 검증 상한)
- 조합 시너지: M1+M4 같은 검증된 조합은 +5.5% 보너스

## 5. SERP 순위대별 효과 (논문 Table 2)

같은 GEO 전략도 검색 결과 페이지 내 사이트 순위에 따라 효과가 다르다.

| 전략 | Rank-1 | Rank-2 | Rank-3 | Rank-4 | Rank-5 |
|------|--------|--------|--------|--------|--------|
| Authoritative | -6.0% | +4.1% | -0.6% | +12.6% | +6.1% |
| Fluency Optimization | -2.0% | +5.2% | +3.6% | -4.4% | +2.2% |
| Cite Sources | -30.3% | +2.5% | +20.4% | +15.5% | **+115.1%** |
| Quotation Addition | -22.9% | -7.0% | +3.5% | +25.1% | +99.7% |
| Statistics Addition | -20.6% | -3.9% | +8.1% | +10.0% | +97.7% |

### 핵심 함의

- **저순위(Rank 4-5) 사이트가 GEO의 최대 수혜자**: Cite Sources +115%, Quotation +99%, Statistics +97%
- **상위 사이트(Rank 1-2)는 일부 모듈에서 오히려 손해**: Cite Sources -30% — 이미 권위적인 사이트는 추가 인용/통계로 인해 비교 우위를 잃는다.
- **GEO는 작은 콘텐츠 제작자에게 기회**: 전통 SEO 시대의 백링크·도메인 권위 격차를 GEO에서는 텍스트 품질로 좁힐 수 있다.

이 스킬은 진단 단계에서 콘텐츠의 추정 SERP 순위대를 가늠해 (사용자가 알려주거나, 도메인 권위에서 추정), 적절한 모듈을 우선순위로 제시한다.

## 6. 한계 — 논문이 직접 명시한 사항 (§9 Limitations)

- 생성형 엔진은 빠르게 진화하므로 GEO 전략도 계속 업데이트 필요
- GEO-bench는 2024년 시점 쿼리 기반. 시간 경과에 따라 변경 가능
- GEO 변경이 SEO 랭킹에 어떤 영향을 주는지는 평가하지 않음
- 영어 기반 실험 — 한국어 등 비영어권에 그대로 일반화하려면 추가 검증 필요

본 스킬은 위 한계를 인정하고, 정기적인 재진단·재변환을 권장한다.

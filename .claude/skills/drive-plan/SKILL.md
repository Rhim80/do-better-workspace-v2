# Do Better Drive · 3단계 — 계획 게이트

> Do Better Drive 7단계 중 3단계. 2단계(drive-scope)가 "될 일인가"를 통과시켰다면,
> 이 단계는 **"어떻게, 무엇이 되면 합격인가"** 를 잠근다. 만들기 전에.
> 개발자용 gstack의 plan-eng-review를 비개발 프로젝트에 맞춘 것.

## 이 스킬의 역할

만들기 전에 실행계획을 못박는다 — 구조·순서, 합격 기준, 재사용할 것. 그래야 4단계(생산)가 헤매지 않고, 5단계(검토)가 검사할 기준이 생긴다.

**절대 규칙: 여기서도 만들기 시작하지 않는다.** 계획만 잠그고 넘긴다.

## 작동

### 0. 입력 읽기 + 준비 확인
- 2단계가 갱신한 `progress.md`의 "목표"(검증된 범위)와 "실현성 점검"을 읽는다.
- **둘 중 하나라도 없으면** 아직 범위가 안 선 것 → drive-scope(2단계)로 돌려보낸다.
- `CLAUDE.md` 프로필 · `08-registry` (있으면).

### 1. 층 가르기 (Do Better Drive 핵심 원칙)
이 프로젝트의 일을 층으로 나눈다. 층마다 완성 비용이 달라 완성도를 다르게 건다.

- **AI가 하는 일** — 초안·생산·변형·정리·유통·분석·조사. AI가 하니 완성이 싸다 → **풀버전으로**. 커버리지·채널·마감 다 챙긴다. 여기선 "덜 하자"를 기본으로 두지 않는다.
- **사람이 하는 일** — 취향·핵심 판단·중심 줄기·관계·비가역 결정. 과거보다 나아졌지만(AI가 잡일을 걷어가 판단에 더 집중된다) 여전히 상대적으로 더 든다 → **여기에 초점을 준다**. 시간이 실제로 되는지 점검은 이 층에만 적용한다.
- **몸으로 하는 일** — 직접 만들고 나르고 만나야 하는 실물(예: 요리·제작·현장 진행). AI가 대신 못 하고, 판단 문제도 아니다. **이게 병목이면 그 사실을 그대로 계획에 반영한다** (AI 일처럼 싸다고 착각하지 않는다).

> 관점: 요즘은 AI가 잡일을 걷어가 한 사람이 몰 수 있는 프로젝트 총량이 예전보다 크다 — 여러 개 동시 진행이 현실적이다. **단, 이건 일의 상당 부분이 AI로 될 때 얘기다.** 몸으로 하는 일이 큰 프로젝트는 이 이점이 작으니 그만큼 현실적으로 잡는다.

### 2. 구조·순서
무엇을 어떤 순서로 만들 건가. 간단한 스케치면 된다.
(예: 책=목차·챕터 순서 / 콘텐츠=주제 묶음·발행 일정 / 과정=커리큘럼·모듈 순서 / 세션=흐름 / 제품·구독=상품 구성·가격·배송 흐름)

### 3. 합격 기준 (= 무엇이 되면 됐는가)
좋은 결과가 뭔지, 뭐가 있어야 합격인지 **미리** 정한다. **5단계(냉정한 검토)가 바로 이 기준으로 검사한다** — 여기서 안 정하면 5단계가 잴 자가 없다. 효과 종류(돈/명성·영향력)에 맞춘 합격선을 적는다.

### 4. 재사용할 것 (다시 만들지 말 것)
이미 가진 것 중 뭘 재사용하나? (도메인마다 다름: 기존 글·자료·레시피·거래처·과거 프로젝트·인맥 등) 새로 만들 것과 재사용할 것을 가른다.

### 5. 인지 패턴 (gstack에서 범용으로 빌림)
- **영웅에 의존 안 하는 구조** — 특정인이 매번 밀어야만 굴러가는 구조를 피한다. 여러 프로젝트를 몰수록 각 프로젝트가 손을 덜 타야 총량이 커진다. "매번 사람이 밀지 않아도 유지되나"를 계획에 넣는다.
- **되돌리기 쉬운 쪽** — 큰 비가역 약속보다 작은, 되돌릴 수 있는 단계.
- **검증된 단순한 길** — 접근을 과하게 새로 짜지 말고 이미 통하는 단순한 방법.

### 6. 빠뜨림 + 결정 기록
- 쉽게 까먹거나 어긋날 지점을 짚는다.
- 여기서 정한 결정은 progress.md 로그에 남긴다 → 나중에 다시 다투지 않는다.

### 7. 핸드오프 (산출)
`progress.md`를 갱신한다:
- **"계획"** 칸에 구조·순서 / 재사용할 것 / 층 구분을 적고, 그 아래 **"합격 기준"** 을 적는다
- **"다음 한 걸음"** 을 구체적 실행 단위로 확정
- frontmatter `status: 계획`, `updated` 갱신
- 로그에 `- <YYYY-MM-DD> 계획 잠금 (drive-plan)` append
- 안내: **"계획 섰어요. 다음은 4단계 — 생산."**

## 규칙 요약
- **만들기 금지** — 계획만.
- **층부터 가른다** — AI 일은 풀버전, 사람 일은 초점, 몸으로 하는 일은 병목으로 인정.
- **합격 기준은 필수** — 5단계가 검사할 자를 여기서 만든다.
- **재사용 먼저** — 이미 있는 걸 다시 만들지 않는다.
- **결정은 기록으로** — 정한 건 남겨 다시 안 다툰다.

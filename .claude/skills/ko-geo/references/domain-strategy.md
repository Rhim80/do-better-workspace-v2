# 도메인별 KoGEO 전략 매핑

논문 §5.1 *Domain-Specific Generative Engine Optimizations* + Table 3 기반.
한국 시장 맥락(B2C 블로그, 상세페이지, 소상공인 등)을 추가 매핑.

## 1. 논문 검증 매핑 (Table 3)

| 논문 메서드 | Top-1 도메인 | Top-2 | Top-3 | KoGEO 모듈 |
|------------|-------------|-------|-------|-----------|
| Authoritative | Debate | History | Science | M8 |
| Fluency Opt. | Business | Science | Health | M4 |
| Cite Sources | Statement | Facts | Law & Gov. | M3 |
| Quotation Addition | People & Society | Explanation | History | M2 |
| Statistics Addition | Law & Gov. | Debate | Opinion | M1 |

### 해석

- **사실/스펙 기반** (Statement, Facts, Law & Gov.) → 출처 표기(M3) + 통계(M1)
- **사람·역사·해설** (People & Society, History, Explanation) → 인용구(M2) + 권위적 어조(M8)
- **비즈니스/과학/헬스** (Business, Science, Health) → 유창성(M4)이 가장 강력
- **토론/정책** (Debate, Opinion) → 권위적 어조(M8) + 통계(M1)

## 3. 도메인 추정 휴리스틱

진단 단계에서 콘텐츠 도메인을 자동 분류할 때 다음 신호를 활용.

| 신호 | 추정 도메인 |
|------|-------------|
| "맛있", "방문", "주차", "예약" 등 | 맛집·카페 |
| "효능", "성분", "피부", "주름", "미백" | 뷰티 |
| "성능", "스펙", "벤치마크", "GPU/CPU" | IT 리뷰 |
| "관련 법률", "조항", "판례", "신청" | 법·세무 |
| "통계청", "식약처", "정부 발표" | 정부·공공 |
| "전문의", "주의사항", "부작용" | 의료 |
| "후기", "별점", "추천" | B2C 리뷰 |
| "ROI", "도입", "도구", "솔루션" | B2B SaaS |
| "역사적", "1900년대", "당시" | 역사 |
| "의견", "주장", "찬반" | 토론·오피니언 |

여러 신호가 겹치면 가장 강한 1~2개 도메인을 선택하고, 해당 도메인의 1~2순위 모듈을 우선 적용한다.

## 4. 도메인 충돌 시 우선순위 결정

한 콘텐츠가 여러 도메인 성격을 가질 때 (예: 의료 + 마케팅 + B2C 리뷰):

1. **타깃 독자가 누구인가** — 일반인이면 M7(쉬운 표현), 전문가면 M6(전문 용어)
2. **콘텐츠의 1차 목적** — 정보 전달이면 M3 우선, 설득이면 M2/M8 우선
3. **검색 진입 단계** — 인지 단계(상단 깔때기) → M10 FAQ, 결정 단계(하단 깔때기) → M1 통계 + M3 출처
4. **콘텐츠 길이** — 짧으면 M5(두괄식) + M1, 길면 M10(FAQ)도 추가 가능

## 5. 도메인 무관 공통 권장

다음 모듈은 거의 모든 도메인에서 적용 가치가 있다 (논문 결과 + 논리적 근거):

- **M5 두괄식 배치** — PAWC 위치 가중치 메커니즘 자체
- **M4 유창성 향상** — 다른 모듈의 효과를 증폭, 시너지 1위 (조합 +5.5%)
- **M9 키워드 과밀 회피** — 적용 안 하면 -8% 역효과 (Stuffing이 누적된 SEO 자산일 경우)

이 3개는 진단 점수가 낮으면 도메인과 무관하게 즉시 적용 후보.

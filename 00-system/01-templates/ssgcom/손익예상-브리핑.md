# 손익예상 브리핑 템플릿 (경영관리팀 FP&A)

> 경영회의 보고용 손익예상 업데이트 지시서. 이 파일을 `@`로 호출하면 아래 양식·검출 규칙·깊이 기준이 그대로 적용된다.
> 작성자: 경영관리팀 → 수신: 경영회의 (본부장·유관 사업부 팀장). 톤: 숫자 옆에 반드시 "그래서 무슨 의미"를 붙인다. 숫자만 있는 줄은 미완성이다.

## 사용 데이터 (반드시 교차)

| 파일 | 역할 | 교차 이유 |
|------|------|----------|
| `매출실적_일별_2026상반기.csv` | OLAP 내려받기 (거래액·순매출·주문건수) | 실적의 원천. 단독으로는 "얼마 팔았나"만 나온다 |
| `카테고리_마스터.csv` | 원가율·수수료율·변동비율·정산기준 | 거래액을 **공헌이익**으로 바꿔주는 열쇠. 직매입(총액)과 위수탁·오픈마켓(수수료 순액)의 정산기준 차이를 반드시 구분 |
| `프로모션_집행내역_2026상반기.csv` | 마케팅팀 제공 (타팀 자료) | 매출 급등이 "번 것"인지 "산 것"인지 판별. 자사분담액을 해당 기간·카테고리 공헌이익에서 차감 |
| `비용계획_2026.csv` | 연간 비용계획 vs 집행 | 공헌이익 아래 고정비를 붙여 영업이익 관점까지 연결 + 집행률 관리 |

## 손익 계산 규칙

- **직매입**: 공헌이익 = 순매출 − (거래액×원가율) − (거래액×물류변동비율) − (거래액×결제수수료율) − 프로모션 자사분담(기간·카테고리 배부)
- **위수탁·오픈마켓**: 공헌이익 = 수수료매출(=순매출) − (거래액×물류변동비율) − (거래액×결제수수료율) − 프로모션 자사분담(배부)
- 프로모션 자사분담액 배부: 프로모션 기간 내 대상 카테고리의 거래액 비중대로 안분
- 거래액(GMV)과 순매출(회계 매출)을 절대 섞지 않는다. 표에는 두 값을 모두 표기하고 어느 기준인지 명시

## 출력 구조

**1. 헤드라인 (KPI 카드 4개)** — 총거래액 / 순매출 / 공헌이익 / 공헌이익률. 각 카드에 전기(전월 또는 전분기) 대비 증감 + "그래서 무슨 의미" 한 줄 필수. 예: "거래액 +9%인데 공헌이익률 −0.8%p — 성장의 질이 나빠지고 있다는 신호".

**2. 사업부별 손익 테이블** — 사업부×월: 거래액, 순매출, 공헌이익, 공헌이익률. 표 아래 사업부마다 "왜 움직였나" 3~4줄 (시즌·프로모션·경쟁 영향·믹스 변화 추정 포함, 추정은 "(추정)" 표기). 1문장으로 뭉개지 말 것.

**3. 프로모션 손익 검증** — 프로모션별로: 기간 중 대상 카테고리 증분 거래액(직전 동일 길이 비프로모션 기간 대비) → 증분 공헌이익 → 자사분담액 차감 → **순효과(+/−)**. "매출은 늘었는데 남는 장사였나"에 하나의 숫자로 답한다. 순효과 음수면 굵게 표시하고 원인 3줄.

**4. 이상 신호 (검출 규칙)** — 아래 규칙에 걸린 항목을 전부 나열, 각 항목에 원인 추정 2~3줄:
- 카테고리 거래액이 직전 2주 평균 대비 **±30% 이상** 변동
- 카테고리 공헌이익률이 전월 대비 **−1.0%p 이상** 하락
- 월 비용 집행액이 계획 대비 **110% 초과** (항목 명시)
- 거래액은 늘고 공헌이익은 줄어든 "성장-이익 역행" 카테고리

**5. 하위 위험군** — 공헌이익 기여 하위 3개 카테고리 + 공통점 1줄. TOP만 보고 끝내지 않는다.

**6. 경영회의 코멘트 후보 3종** — 회의에서 그대로 읽을 수 있는 문장으로: ① 숫자형 (핵심 수치 중심 1문장) ② 흐름형 (분기 추세 서사 1~2문장) ③ 헤드라인형 (한 줄 결론). 자리 비워두지 말고 미리 뽑아서 고르게 한다.

**7. 액션 테이블 (4열)** — | 담당(팀 단위) | 액션 | 기한 | 우선순위 | + 각 행에 성공 기준 1줄. "검토 필요" 같은 뭉갠 액션 금지 — 무엇을 얼마까지.

## 길이·형식 제약

- 마크다운 기준 A4 2장 이내. 단, **해석·원인 줄은 생략 금지** — 줄이려면 표의 행을 줄이지 해석을 줄이지 않는다
- 금액 단위: 억원 (소수 1자리). 비율: %p 표기 구분
- 모든 수치는 CSV에서 계산 가능해야 하며, 근거 없는 수치 창작 금지
